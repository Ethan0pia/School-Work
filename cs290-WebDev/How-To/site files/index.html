Introduction:

Today, we will be teaching you about APIs. Specifically, we will discuss the Deck of Cards API and how to use it to make a web-based Blackjack game.

What is an API?

An API is an "Application Program Interface". This is a set of routines that can be called by the user to do various things. Many require the user to sign up for an API ID to access them. You should never share your ID online since other users could use it and possibly cause your ID to be banned. APIs generally have a limit of calls the user can make in a set period. This is meant to stop the API server from being bogged down by too many calls trying to access the API repeatedly in quick succession. Try to limit your calls to an API to as few per second as needed.

What kind of APIs are there?

There are many APIs available for use. Some APIs return weather information when sent a zip code or city. Others, like the NASA API, can send back images that are stored on a database. Deck of Cards API holds all the tools needed to create most card games, which we will discuss on the next page.
